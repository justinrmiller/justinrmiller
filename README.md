## 🚀 Featured Projects

### 1. [ml-homelab](https://github.com/justinrmiller/ml-homelab)

A local development environment for learning how to train models and support machine learning workflows using Ray and Streamlit. Provides a self-contained setup for experimenting with distributed training and ML pipelines on local hardware before scaling to the cloud.

**Tech Stack**: Python, Ray, Streamlit

**Key Features**:
- Local environment for model training and ML workflows
- Distributed compute with Ray
- Interactive interfaces and dashboards via Streamlit

### 2. [ray-fiftyone](https://github.com/justinrmiller/ray-fiftyone)

A Ray Data integration for FiftyOne datasets, enabling distributed processing of computer vision datasets using Ray's parallel execution framework. Bridges Ray's distributed compute with FiftyOne's dataset management for large-scale vision workflows.

**Tech Stack**: Python, Ray, FiftyOne

**Key Features**:
- Distributed reading and writing of FiftyOne datasets
- Cloud storage support (S3, GCS, Azure Blob)
- Remote MongoDB / FiftyOne Atlas connectivity
- Field selection, view stages, and label serialization
- Export to Parquet for data warehousing

### 3. [ray-vector-io](https://github.com/justinrmiller/ray-vector-io)

Ray Vector Database Input/Output Utilities. This project provides tools for seamless integration between Ray, a distributed computing framework, and various vector databases. Designed to optimize data pipeline workflows involving vector embeddings for AI applications like semantic search and recommendation systems.

**Tech Stack**: Python, Ray

**Key Features**:
- Streamlined data loading/extraction from vector databases
- Integration with Ray's distributed computing ecosystem
- Support for various vector database formats

### 4. [opensearch-lancedb-migration](https://github.com/justinrmiller/opensearch-lancedb-migration)

A benchmark and migration comparison between OpenSearch and LanceDB for vector search. Evaluates the tradeoffs of moving vector workloads between a traditional search engine and a columnar, embedded vector store.

**Tech Stack**: Python, OpenSearch, LanceDB

**Key Features**:
- Side-by-side comparison of OpenSearch and LanceDB
- Vector search benchmarking
- Migration patterns between vector backends

### 5. [ray-qdrant-embeddings](https://github.com/justinrmiller/ray-qdrant-embeddings)

A scalable semantic search system for news articles using Ray for distributed embedding generation, Qdrant for vector search, and Streamlit for the user interface. Demonstrates an end-to-end pipeline from raw text to interactive semantic search.

**Tech Stack**: Python, Ray, Qdrant, Streamlit

**Key Features**:
- Distributed embedding generation with Ray Actors
- Vector storage and querying with Qdrant
- Interactive search UI built with Streamlit
- Pre-configured for the AG News dataset (120K+ articles)

### 6. [spark-kafka-parquet-example](https://github.com/justinrmiller/spark-kafka-parquet-example)

An example project demonstrating how to combine Apache Spark Streaming, Kafka, and Parquet to build a robust data pipeline. This project transforms JSON objects streamed over Kafka into Parquet files stored in S3, showcasing a complete ETL workflow.

**Tech Stack**: Scala, Apache Spark, Kafka, Parquet, S3

**Key Features**:
- Real-time data processing with Spark Streaming
- JSON message consumption from Kafka topics
- Efficient storage using Parquet file format
- Cloud integration with Amazon S3
